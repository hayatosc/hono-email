---
title: render()
description: The primary runtime API for rendering email from hono/jsx.
---

`render()` is the primary runtime API. It turns a `hono/jsx` tree into HTML email, plain text, and a list of warnings.

## Return value

`render()` resolves to an object with the following shape:

```ts
{
  html: string
  text: string
  warnings: string[]
}
```

- `html` — the rendered and normalized HTML email body.
- `text` — the plain-text version derived from the HTML.
- `warnings` — compatibility warnings collected during rendering.

## Basic usage

```tsx
const { html, text, warnings } = await render(<WelcomeEmail />)
```

Strict mode is enabled by default. If the JSX contains markup or CSS that is risky for email clients, `render()` throws or warns depending on the configuration.

## Common options

```tsx
const { html, text } = await render(<WelcomeEmail />, {
  pretty: true,
  widows: true,
  onWarning: 'error',
  text: {
    headingStyle: 'preserve',
    linkFormat: 'text-only',
  },
})
```

- `strict` — enables strict email validation (default: `true`).
- `pretty` — pretty-prints the HTML.
- `minify` — minifies the HTML (default: `true`, ignored when `pretty` is set).
- `widows` — joins the last two words of each text block with a non-breaking space.
- `onWarning` — controls how warnings are surfaced (`'warn'`, `'error'`, `'silent'`, or a callback).
- `text` — plain-text conversion options.

See the [API Reference](/api/core/index/functions/render/) for the full list of options.

## Warning handling

Use `onWarning` to control how warnings are handled:

- `'warn'` — logs warnings with `console.warn` and returns them in `warnings`.
- `'error'` — throws when any warning is collected.
- `'silent'` — returns warnings without logging them.
- `callback` — calls a custom function for each warning.

## Color expansion

`render()` always expands three-digit hex colors to six-digit hex inside `style` attributes and `<style>` blocks:

```css
/* input */
color: #abc;

/* output */
color: #aabbcc;
```
